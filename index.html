Investigation Writeups

WR_001
Amadey Malware — Memory Forensics
Full memory dump analysis of an Amadey-infected Windows 7 workstation using Volatility 3. Identified the malicious process masquerading as lssass.exe, extracted the C2 server IP, traced downloaded payloads, and mapped persistence mechanisms.
Memory Forensics Malware Analysis Volatility 3 Windows Forensics C2 Detection T1055 · T1036
→
Overview
This lab focuses on endpoint forensics using Volatility 3 on a Windows 7 memory dump from a machine infected with the Amadey malware — a modular info-stealer known for credential theft, persistence, and downloading additional payloads. The goal was to identify the malicious process, extract C2 communications, and map persistence mechanisms.
Investigation Steps
  1. 01Listed all running processes using windows.pslist to identify the suspicious process masquerading as a legitimate Windows binary. A process named lssass.exe (note the double 's') was identified — spoofing the legitimate lsass.exe to evade detection.
  2. 02Used windows.cmdline with the suspicious PID to extract the full file path. The process was executing from an unusual temp directory rather than System32, confirming it was not a legitimate Windows process.
  3. 03Ran windows.netscan to extract active and closed network connections from memory. Identified the C2 server IP address associated with the malicious process making outbound connections.
  4. 04Used windows.memmap to dump the memory region of the malicious process, then analyzed the dump with strings to extract HTTP GET requests — revealing the distinct files being fetched from the C2 server.
  5. 05Used windows.filescan filtered by the malware filename to identify full file paths on disk — locating where downloaded components were stored for execution.
  6. 06Mapped the process tree using windows.pstree to identify the child process spawned by lssass.exe to execute the downloaded payloads.
  7. 07Scanned for additional persistence locations using windows.filescan filtered by the malware name — discovering secondary file locations used by Amadey to maintain persistence across reboots.
Key Volatility Commands
vol -f 'Windows 7 x64-Snapshot4.vmem' windows.pslist vol -f 'Windows 7 x64-Snapshot4.vmem' windows.cmdline --pid <PID> vol -f 'Windows 7 x64-Snapshot4.vmem' windows.netscan vol -f 'Windows 7 x64-Snapshot4.vmem' windows.memmap --pid <PID> --dump vol -f 'Windows 7 x64-Snapshot4.vmem' windows.filescan | grep -E "<FILE_NAME>" vol -f 'Windows 7 x64-Snapshot4.vmem' windows.pstree vol -f 'Windows 7 x64-Snapshot4.vmem' windows.filescan | grep "lssass" -E

WR_002
Silent Breach — Disk Forensics & Malware Decryption
Disk image forensics investigation using FTK Imager. Traced a malicious EXE download through browser history, extracted C2 IP addresses from Windows Mail artifacts, decoded an obfuscated PowerShell ransomware script, and successfully decrypted AES-256-CBC encrypted files using derived key and IV via OpenSSL.
Disk Forensics Malware Decryption FTK Imager CyberChef OpenSSL PowerShell Analysis T1486 · T1140
→
Overview
A disk forensics investigation on a Windows image belonging to a user named Ethan. The investigation required tracing the download of a malicious executable, analyzing browser artifacts, extracting C2 IP addresses from Windows Mail data, reversing an obfuscated PowerShell encryption script, and decrypting AES-256-CBC encrypted files to recover the flag.
Investigation Steps
  1. 01Imported the disk image into FTK Imager and located the malicious EXE file. Extracted the MD5 hash directly from FTK Imager's file properties for verification against threat intel databases.
  2. 02Identified the download URL by examining the Zone.Identifier (ZoneTransfer) alternate data stream attached to the malicious file — this Windows security feature records the URL source of downloaded files.
  3. 03Determined the download application by extracting and analyzing SQLite browser history databases from both Chrome and Microsoft Edge at their respective AppData paths using DB Browser for SQLite.
  4. 04Extracted the Windows Mail artifact file HxStore.hxd from the WindowsCommunicationsApps package directory. Analyzed it using Notepad++ with the HEX Editor plugin to extract three C2 IP addresses mentioned in an email.
  5. 05Used the Strings tool on the malicious EXE to extract a reversed Base64-encoded PowerShell script. Decoded it using two methods: CyberChef (Reverse + From Base64 recipe) and Linux echo command — revealing the encryption password and AES parameters.
  6. 06Reproduced the PBKDF2 key derivation process from the PowerShell script to extract the exact AES-256-CBC Key and IV values used for encryption — since OpenSSL could not derive them without replicating the original algorithm.
  7. 07Decrypted the encrypted PDF files (IMF-Secret.enc and IMF-Mission.enc) using OpenSSL with the derived key and IV, verified integrity with the file command, and recovered the secret flag from the decrypted PDFs.
Key Artifact Paths
Chrome History: \AppData\Local\Google\Chrome\User Data\Default\History Edge History: \AppData\Local\Microsoft\Edge\User Data\Default\History Windows Mail: \AppData\Local\Packages\microsoft.windowscommunicationsapps_8wekyb3d8bbwe\LocalState\HxStore.hxd Encrypted Files: C:\Users\ethan\Desktop\*.enc
Decryption Method
# Step 1 — Reproduce key derivation (PBKDF2 / RFC2898) # Password and salt extracted from reversed Base64 PowerShell script # Algorithm: AES-256-CBC, iterations from script parameters # Step 2 — Decrypt using OpenSSL with derived key and IV openssl enc -d -aes-256-cbc -K <hex_key> -iv <hex_iv> -in IMF-Secret.enc -out IMF-Secret.pdf # Step 3 — Verify decryption file IMF-Secret.pdf

Key Lesson
Malware authors who use hardcoded passwords and static salts in encryption routines create recoverable ciphertexts. By extracting the script, reversing the Base64 obfuscation, and replicating the PBKDF2 key derivation, the encryption was fully reversed — highlighting the importance of randomized cryptographic parameters in secure implementations.
